# Риски и митигация — PetFeed

## Матрица оценки рисков

**Вероятность:** 1 (низкая) → 3 (высокая)  
**Влияние:** 1 (низкое) → 3 (критическое)  
**Приоритет:** Вероятность × Влияние

---

## Бизнес-риски

### BR-001: Низкий retention пользователей
| Параметр | Значение |
|---|---|
| **Описание** | Пользователи создают профиль, получают рацион и уходят — нет причины возвращаться |
| **Вероятность** | 3 (высокая) |
| **Влияние** | 3 (критическое) |
| **Приоритет** | 9 🔴 |

**Митигация:**
- Ежедневные напоминания о кормлении — причина заходить каждый день
- Геймификация: стрики кормления, бейджи ("Идеальный хозяин 7 дней")
- Еженедельные советы по питанию в рассылке бота
- Напоминания о взвешивании питомца раз в неделю

---

### BR-002: Сложность монетизации на старте
| Параметр | Значение |
|---|---|
| **Описание** | Партнёры не хотят интегрироваться без аудитории, аудитория не растёт без партнёров |
| **Вероятность** | 2 (средняя) |
| **Влияние** | 2 (среднее) |
| **Приоритет** | 4 🟡 |

**Митигация:**
- Начать с ручной интеграции: просто ссылка на сайт партнёра без API
- Первые 2–3 партнёра на личных договорённостях
- Показывать партнёру не MAU, а конверсию намерения купить

---

### BR-003: Конкуренция со стороны крупных платформ
| Параметр | Значение |
|---|---|
| **Описание** | Petshop, Wildberries или ВКонтакте могут запустить похожий функционал |
| **Вероятность** | 1 (низкая на старте) |
| **Влияние** | 3 (критическое) |
| **Приоритет** | 3 🟢 |

**Митигация:**
- Глубокая специализация на экзотике (рептилии, грызуны) — крупные платформы туда не пойдут
- Строить сообщество и лояльность раньше, чем придут крупные игроки
- Данные пользователей — наш ров: персонализация улучшается с каждым взаимодействием

---

### BR-004: Недоверие к рекомендациям бота
| Параметр | Значение |
|---|---|
| **Описание** | Пользователи не доверяют советам AI/бота по здоровью питомца |
| **Вероятность** | 2 (средняя) |
| **Влияние** | 2 (среднее) |
| **Приоритет** | 4 🟡 |

**Митигация:**
- Верификация контента ветеринарами (указывать имена в профиле)
- Все рекомендации со ссылками на источники
- Дисклеймер: "Не заменяет визит к ветеринару"
- Раздел "Одобрено ветеринаром" как социальное доказательство

---

## Технические риски

### TR-001: Неточность рекомендаций для редких животных
| Параметр | Значение |
|---|---|
| **Описание** | Недостаточно данных в базе знаний для экзотических пород и видов |
| **Вероятность** | 3 (высокая) |
| **Влияние** | 2 (среднее) |
| **Приоритет** | 6 🟠 |

**Митигация:**
- Запускать поэтапно: сначала кошки/собаки с полной базой, затем добавлять виды
- Для редких животных — флаг "Базовые рекомендации, уточните у специалиста"
- Партнёрство с ветеринарными специалистами по экзотике для наполнения базы

---

### TR-002: Нестабильность LLM API
| Параметр | Значение |
|---|---|
| **Описание** | Сбои или деградация Claude/GPT API влияют на AI-ассистента |
| **Вероятность** | 2 (средняя) |
| **Влияние** | 2 (среднее) |
| **Приоритет** | 4 🟡 |

**Митигация:**
- Кэширование популярных вопросов и ответов
- Fallback на структурированную базу знаний без LLM
- Мониторинг доступности API, алерты при деградации

---

### TR-003: Блокировка Telegram-бота
| Параметр | Значение |
|---|---|
| **Описание** | Telegram может ограничить работу бота (спам-жалобы, нарушение правил) |
| **Вероятность** | 1 (низкая) |
| **Влияние** | 3 (критическое) |
| **Приоритет** | 3 🟢 |

**Митигация:**
- Соблюдать правила Telegram Bot API (лимиты сообщений, без спама)
- Параллельно развивать веб-версию как резервный канал
- Собирать email/контакты пользователей для коммуникации вне Telegram

---

### TR-004: Утечка или потеря данных пользователей
| Параметр | Значение |
|---|---|
| **Описание** | Компрометация базы данных с профилями питомцев и историей заказов |
| **Вероятность** | 1 (низкая) |
| **Влияние** | 3 (критическое) |
| **Приоритет** | 3 🟢 |

**Митигация:**
- Шифрование данных в покое и при передаче
- Минимальный сбор персональных данных (только Telegram ID, не ФИО)
- Регулярные бэкапы, разграничение прав доступа

---

## Операционные риски

### OR-001: Уход ключевого партнёра
| Параметр | Значение |
|---|---|
| **Описание** | Основной партнёр по доставке прекращает сотрудничество |
| **Вероятность** | 2 (средняя) |
| **Влияние** | 2 (среднее) |
| **Приоритет** | 4 🟡 |

**Митигация:**
- Минимум 2–3 партнёра с первых месяцев
- Не строить критическую зависимость от одного партнёра
- Контракты с минимальным периодом уведомления об уходе

---

### OR-002: Медленный рост аудитории
| Параметр | Значение |
|---|---|
| **Описание** | Органический рост идёт медленнее прогноза, маркетинговый бюджет исчерпан |
| **Вероятность** | 2 (средняя) |
| **Влияние** | 2 (среднее) |
| **Приоритет** | 4 🟡 |

**Митигация:**
- Валидировать спрос через кастодев до запуска
- A/B тестировать разные форматы посевов
- Pivot в нишу (например, только рептилии) при слабом охвате широкой аудитории
