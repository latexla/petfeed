# Breed Recognition — Design Spec

**Date:** 2026-04-25
**Status:** Approved

---

## Цель

Точно определять породу питомца даже при опечатках или вводе через фото. Хранить нераспознанные породы для пополнения реестра.

---

## Скоуп

- Виды: **собаки и кошки** (Фаза 1)
- Текстовый ввод: fuzzy-matching через RapidFuzz
- Фото-ввод: DeepSeek Vision → нормализация через fuzzy

---

## Архитектура

```
breed_registry (DB)
      ↓
BreedRepository  ←  RapidFuzz matching
      ↓
BreedService  ←  DeepSeek Vision (для фото)
      ↓
bot/handlers/pet_creation.py
```

---

## Модель данных

### Таблица `breed_registry`
| Колонка | Тип | Описание |
|---|---|---|
| id | SERIAL PK | — |
| canonical_name | VARCHAR(100) | Официальное название EN |
| canonical_name_ru | VARCHAR(100) | Официальное название RU |
| species | VARCHAR(50) | `dog` / `cat` |
| aliases | TEXT[] | Варианты написания: ["джек расел", "JRT", ...] |

Fuzzy-matching выполняется против `canonical_name + canonical_name_ru + aliases`.

---

## Логика матчинга (текст)

| Score | Действие |
|---|---|
| ≥ 85% | Тихо сохраняем `canonical_name`, пользователь не замечает |
| 60–84% | Показываем топ-3 кнопками + кнопка "Сохранить как введено" |
| < 60% | "Порода не найдена" → [Ввести заново] [Сохранить как введено] |

**"Сохранить как введено"** — сохраняет сырой ввод пользователя в `pets.breed`. Позволяет в admin panel видеть популярные нераспознанные породы и добавлять их в реестр.

---

## Логика распознавания по фото

1. Пользователь нажимает **[Отправить фото 📷]** на шаге породы
2. Бот запрашивает фото
3. Скачиваем файл через Telegram Bot API
4. Отправляем в DeepSeek Vision с промптом:
   > "What breed is this dog/cat? Answer with only the breed name in English."
5. Ответ прогоняем через fuzzy-matching (те же правила, что для текста)
6. Показываем результат как кнопку для подтверждения
