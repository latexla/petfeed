# Acceptance Criteria — PetFeed

## AC-001: Создание профиля питомца
**Связь:** US-001, UC-001

**Описание:** Пользователь успешно создаёт профиль питомца через пошаговый диалог в боте.

**Предусловия:**
- Пользователь запустил бота командой /start
- Профиль питомца ещё не создан

**Условия проверки:**
- Бот предлагает выбор вида животного из списка: кошка, собака, грызун, птица, рептилия
- При выборе вида отображается список пород (или "Метис / Не знаю")
- Поля возраст и вес принимают только числовые значения в допустимом диапазоне
- При вводе некорректного значения бот выводит сообщение об ошибке и повторяет запрос
- После заполнения всех полей отображается сводка профиля с кнопкой подтверждения
- Профиль сохраняется в БД не более чем за 2 секунды
- После сохранения бот предлагает перейти к рекомендациям по питанию

**Постусловия:**
- Профиль питомца сохранён в базе данных
- Пользователь видит главное меню с активными разделами

**Приоритет:** Критический  
**Обоснование:** Без профиля невозможна персонализация — ключевая ценность продукта

**Методы проверки:** Ручное тестирование, E2E-тест диалога  
**Критерии успешности:** 100% полей заполнено, профиль сохранён, переход к меню выполнен

## AC-002: Получение рекомендаций по питанию
**Связь:** US-004, US-006, US-007, UC-002

**Описание:** Система формирует персональный рацион на основе данных профиля питомца.

**Предусловия:**
- Профиль питомца создан (AC-001 пройден)
- База знаний содержит данные для выбранного вида животного

**Условия проверки:**
- Рацион содержит: разрешённые продукты, запрещённые продукты, частоту и объём кормления
- Суточная норма калорий рассчитана с учётом веса, возраста и выбранной цели
- Для редкого вида/породы выводится пометка "Уточните у ветеринара"
- При весе питомца за пределами нормы (±30% от стандарта для породы) отображается предупреждение
- Рекомендации отображаются не более чем за 3 секунды
- После получения рациона доступны кнопки: "Заказать корм", "Настроить напоминания", "Задать вопрос"

**Постусловия:**
- Рацион сохранён в профиле питомца
- Дата последнего обновления рациона зафиксирована

**Приоритет:** Критический  
**Обоснование:** Основная ценность продукта — персональные рекомендации

**Методы проверки:** Ручное тестирование, сравнение с эталонными данными по видам  
**Критерии успешности:** Рацион содержит все 3 блока, время ответа ≤3 сек, данные соответствуют виду питомца

## AC-003: Стоп-лист продуктов
**Связь:** US-005, UC-005

**Описание:** Пользователь получает список запрещённых продуктов для своего питомца.

**Предусловия:**
- Профиль питомца создан

**Условия проверки:**
- Стоп-лист загружается автоматически на основе вида питомца из профиля
- Каждый продукт имеет пометку степени опасности: ⚠️ умеренно / 🚫 опасно / ☠️ смертельно
- Каждый продукт сопровождается кратким пояснением (1–2 предложения)
- Функция поиска продукта по стоп-листу возвращает результат не более чем за 1 секунду
- Если продукт не найден в базе — выводится сообщение "Уточните у ветеринара"
- Список содержит минимум 10 позиций для кошек и собак, минимум 5 для других видов

**Постусловия:**
- Пользователь ознакомлен со стоп-листом

**Приоритет:** Высокий  
**Обоснование:** Безопасность питомца — критична для доверия к продукту

**Методы проверки:** Ручное тестирование, верификация списка с ветеринарными источниками  
**Критерии успешности:** Все продукты имеют степень опасности, поиск работает корректно

## AC-004: Настройка напоминаний о кормлении
**Связь:** US-008, UC-003

**Описание:** Пользователь настраивает расписание напоминаний и получает уведомления в указанное время.

**Предусловия:**
- Профиль питомца создан
- Рацион сформирован (частота кормления известна)

**Условия проверки:**
- Система предлагает расписание на основе рекомендованной частоты кормления
- Пользователь может принять предложенное или задать своё расписание
- Время вводится в формате HH:MM; при некорректном вводе — ошибка и повтор запроса
- Максимум 6 напоминаний в сутки на одного питомца
- Уведомление приходит в течение ±1 минуты от указанного времени
- Пользователь может отключить все напоминания одной командой

**Постусловия:**
- Расписание сохранено в БД
- Уведомления активированы через Telegram

**Приоритет:** Высокий  
**Обоснование:** Напоминания — основной механизм ежедневного возврата пользователя

**Методы проверки:** Ручное тестирование, автотест точности времени уведомлений  
**Критерии успешности:** Уведомление приходит с точностью ±1 мин, отключение работает мгновенно

## AC-005: Заказ корма через партнёра
**Связь:** US-011, UC-004

**Описание:** Пользователь переходит к партнёру для заказа рекомендованного корма.

**Предусловия:**
- Рацион сформирован
- Хотя бы один партнёр интегрирован и доступен

**Условия проверки:**
- Список кормов отображается с ценой, объёмом и названием партнёра
- При недоступности партнёра (timeout >5 сек) выводится сообщение об ошибке
- При отсутствии рекомендованного корма у партнёра отображаются альтернативы
- Реферальная ссылка формируется уникальная для каждого перехода
- Переход по ссылке фиксируется в системе не более чем за 1 секунду
- После перехода в истории пользователя появляется запись о переходе

**Постусловия:**
- Реферальная сессия зафиксирована в БД
- При получении webhook — заказ помечается как "Подтверждён"

**Приоритет:** Критический  
**Обоснование:** Основной источник дохода на старте

**Методы проверки:** Ручное тестирование, проверка webhook от партнёра в тестовой среде  
**Критерии успешности:** Ссылка уникальна, переход зафиксирован, история обновлена

## AC-006: Обновление веса питомца
**Связь:** US-010, UC-006

**Описание:** Пользователь вносит новый вес питомца и получает обновлённые рекомендации.

**Предусловия:**
- Профиль питомца создан
- Есть хотя бы одна предыдущая запись веса

**Условия проверки:**
- Принимается не более одной записи веса в день
- При изменении веса на ±5% система пересчитывает нормы питания и уведомляет пользователя
- При изменении веса на ±10% и более — выводится предупреждение с рекомендацией обратиться к ветеринару
- Динамика веса отображается в формате "+X г (+Y%)" по сравнению с предыдущим значением
- Вес за пределами физиологической нормы для вида (например, кошка >15 кг) — предупреждение

**Постусловия:**
- Новое значение веса сохранено с датой записи
- Рацион пересчитан при отклонении ≥5%

**Приоритет:** Средний  
**Обоснование:** Важно для долгосрочного мониторинга здоровья питомца

**Методы проверки:** Ручное тестирование граничных значений  
**Критерии успешности:** Пересчёт при ≥5%, предупреждение при ≥10%, одна запись в день

## AC-007: AI-ассистент по питанию
**Связь:** US-014, UC-007

**Описание:** Пользователь задаёт вопрос о питании питомца и получает персонализированный ответ.

**Предусловия:**
- Профиль питомца создан
- Лимит запросов не исчерпан (≤10 в день для бесплатного тарифа)

**Условия проверки:**
- Ответ на вопрос приходит не более чем за 10 секунд
- Ответ учитывает вид, породу, возраст и вес питомца из профиля
- Каждый ответ содержит дисклеймер "Не является ветеринарной консультацией"
- При вопросе не по теме питания — бот вежливо перенаправляет в контекст питомца
- При исчерпании лимита (10 запросов/день) — предложение Premium с описанием преимуществ
- При недоступности LLM API — ответ из кэша или сообщение об ошибке

**Постусловия:**
- Вопрос и ответ сохранены в истории диалога
- Счётчик запросов увеличен на 1

**Приоритет:** Высокий  
**Обоснование:** AI-ассистент — ключевое конкурентное преимущество

**Методы проверки:** Ручное тестирование, нагрузочный тест API  
**Критерии успешности:** Ответ ≤10 сек, дисклеймер присутствует, лимит соблюдается

## AC-008: Подключение партнёра-зоомагазина
**Связь:** US-017, UC-008

**Описание:** Партнёр успешно интегрирует свой каталог в платформу PetFeed.

**Предусловия:**
- Договор с партнёром подписан
- Администратор подтвердил заявку

**Условия проверки:**
- Система генерирует уникальный API-ключ для партнёра
- Тестовый запрос к каталогу партнёра выполняется успешно (HTTP 200)
- При недоступности каталога партнёра (timeout >10 сек) — интеграция помечается как неактивная
- Товары партнёра появляются в рекомендациях не позднее чем через 1 час после интеграции
- Webhook от партнёра обрабатывается системой в течение 5 секунд

**Постусловия:**
- Партнёр помечен как активный в системе
- Товары доступны пользователям

**Приоритет:** Высокий  
**Обоснование:** Без партнёров нет монетизации

**Методы проверки:** Интеграционное тестирование с тестовым каталогом  
**Критерии успешности:** API-ключ сгенерирован, товары отображаются, webhook обрабатывается
