# Use Cases — PetFeed

## Акторы системы

| Актор | Тип | Описание |
|---|---|---|
| Владелец питомца | Основной | Пользователь Telegram, взаимодействует с ботом |
| Telegram Bot API | Вторичный | Платформа доставки сообщений |
| PetFeed Backend | Вторичный | Сервер бизнес-логики |
| База знаний (nutrition DB) | Вторичный | Хранилище данных о питании животных |
| Партнёр (зоомагазин) | Вторичный | Внешняя система для оформления заказов |
| Администратор | Вторичный | Управляет контентом платформы |

## UC-001: Создание профиля питомца

| Элемент | Описание |
|---|---|
| **Название** | UC-001. Создание профиля питомца |
| **User Story** | US-001: Как новый пользователь, я хочу создать профиль питомца, чтобы получать персональные рекомендации |
| **Участники** | Основной: Владелец питомца. Вторичные: Telegram Bot API, PetFeed Backend |
| **Предусловия** | Пользователь запустил бота командой /start |
| **Ограничения** | В MVP — не более 1 питомца на аккаунт |
| **Триггер** | Пользователь нажимает кнопку "Добавить питомца" |

**Основной сценарий:**
1. Система выводит список видов животных (кошка, собака, грызун, птица, рептилия)
2. Пользователь выбирает вид животного
3. Система запрашивает породу (или "Не знаю / Метис")
4. Пользователь вводит породу
5. Система запрашивает имя питомца
6. Пользователь вводит имя
7. Система запрашивает возраст (в месяцах или годах)
8. Пользователь вводит возраст
9. Система запрашивает текущий вес (в кг)
10. Пользователь вводит вес
11. Система сохраняет профиль и выводит сводку данных
12. Система предлагает перейти к рекомендациям по питанию

**Альтернативные сценарии:**
- **п.4: Порода не найдена** → Система предлагает ввести вручную или выбрать "Другое"
- **п.7–9: Некорректный формат ввода** → Система сообщает об ошибке и повторяет запрос
- **п.11: Ошибка сохранения (500)** → Система сообщает "Попробуйте позже", данные не теряются

**Постусловия:** Профиль питомца сохранён в БД, пользователь готов получать рекомендации

## UC-002: Получение рекомендаций по питанию

| Элемент | Описание |
|---|---|
| **Название** | UC-002. Получение персональных рекомендаций по питанию |
| **User Story** | US-004: Как владелец питомца, я хочу получить персональный рацион, чтобы кормить животное правильно |
| **Участники** | Основной: Владелец питомца. Вторичные: PetFeed Backend, База знаний |
| **Предусловия** | Профиль питомца создан (UC-001) |
| **Ограничения** | Рекомендации формируются только на основе верифицированных данных из базы знаний |
| **Триггер** | Пользователь нажимает "Рацион питания" в главном меню |

**Основной сценарий:**
1. Система загружает профиль активного питомца
2. Система запрашивает текущую цель (поддержание веса / похудение / набор веса / рост)
3. Пользователь выбирает цель
4. Система обращается к базе знаний по параметрам: вид, порода, возраст, вес, цель
5. Система рассчитывает суточную норму калорий и объём порций
6. Система формирует рекомендации: что можно кормить, что нельзя, как часто
7. Система выводит рацион в структурированном виде (разрешено / запрещено / расписание)
8. Система предлагает кнопки: "Заказать корм", "Настроить напоминания", "Задать вопрос"

**Альтернативные сценарии:**
- **п.4: Животное редкого вида/породы** → Система выводит базовые рекомендации для вида с пометкой "Уточните у ветеринара"
- **п.4: Данные в базе отсутствуют** → Система предлагает задать вопрос AI-ассистенту (UC-007)
- **п.5: Вес питомца в критической зоне** → Система добавляет предупреждение и рекомендует обратиться к ветеринару

**Постусловия:** Пользователь получил персонализированный рацион, сохранённый в его профиле

## UC-003: Настройка напоминаний о кормлении

| Элемент | Описание |
|---|---|
| **Название** | UC-003. Настройка напоминаний о кормлении |
| **User Story** | US-008: Как владелец питомца, я хочу настроить напоминания, чтобы не пропускать кормление |
| **Участники** | Основной: Владелец питомца. Вторичные: Telegram Bot API, PetFeed Backend |
| **Предусловия** | Профиль питомца создан, рацион сформирован |
| **Ограничения** | Не более 6 напоминаний в сутки на одного питомца |
| **Триггер** | Пользователь нажимает "Настроить напоминания" |

**Основной сценарий:**
1. Система отображает рекомендуемое расписание кормлений на основе рациона
2. Пользователь подтверждает расписание или выбирает "Настроить вручную"
3. Система запрашивает время каждого кормления
4. Пользователь вводит время (формат HH:MM)
5. Система запрашивает часовой пояс пользователя
6. Пользователь указывает часовой пояс
7. Система сохраняет расписание и активирует напоминания
8. Система подтверждает: "Напоминания настроены" с перечнем времени

**Альтернативные сценарии:**
- **п.4: Некорректный формат времени** → Система просит повторить ввод
- **п.7: Telegram заблокировал уведомления** → Система информирует пользователя о необходимости разрешить уведомления
- **п.2: Пользователь выбирает "Выключить напоминания"** → Система деактивирует все напоминания для питомца

**Постусловия:** Расписание напоминаний сохранено, уведомления активированы

## UC-004: Заказ корма через партнёра

| Элемент | Описание |
|---|---|
| **Название** | UC-004. Заказ рекомендованного корма |
| **User Story** | US-011: Как владелец питомца, я хочу заказать корм одной кнопкой, чтобы сэкономить время |
| **Участники** | Основной: Владелец питомца. Вторичные: PetFeed Backend, Партнёр (зоомагазин) |
| **Предусловия** | Пользователь получил рекомендации по корму (UC-002), партнёр интегрирован |
| **Ограничения** | Доступность товара зависит от партнёра; PetFeed не обрабатывает платёж напрямую |
| **Триггер** | Пользователь нажимает "Заказать корм" |

**Основной сценарий:**
1. Система отображает список рекомендованных кормов с ценами от партнёра
2. Пользователь выбирает корм
3. Система отображает детали: состав, объём, цена, партнёр
4. Пользователь нажимает "Перейти к заказу"
5. Система формирует реферальную ссылку и перенаправляет на сайт/приложение партнёра
6. Пользователь оформляет заказ на стороне партнёра
7. Партнёр уведомляет PetFeed об успешном заказе (webhook)
8. Система фиксирует заказ в истории пользователя и начисляет комиссию

**Альтернативные сценарии:**
- **п.1: Рекомендованный корм отсутствует у партнёра** → Система предлагает альтернативные позиции
- **п.5: Партнёр недоступен (503)** → Система уведомляет пользователя и предлагает другого партнёра
- **п.7: Webhook не получен в течение 24ч** → Система помечает заказ как "ожидает подтверждения"

**Постусловия:** Переход на страницу партнёра выполнен, реферальная сессия зафиксирована

## UC-005: Просмотр стоп-листа продуктов

| Элемент | Описание |
|---|---|
| **Название** | UC-005. Просмотр запрещённых продуктов |
| **User Story** | US-005: Как владелец питомца, я хочу видеть стоп-список, чтобы не навредить питомцу |
| **Участники** | Основной: Владелец питомца. Вторичные: PetFeed Backend, База знаний |
| **Предусловия** | Профиль питомца создан |
| **Ограничения** | Список основан только на верифицированных ветеринарных источниках |
| **Триггер** | Пользователь нажимает "Что нельзя давать" в меню |

**Основной сценарий:**
1. Система загружает вид и породу питомца из профиля
2. Система обращается к базе знаний за стоп-листом для данного вида
3. Система выводит список запрещённых продуктов с пояснением опасности
4. Каждый продукт снабжён пометкой степени опасности (⚠️ умеренно / 🚫 опасно / ☠️ смертельно)
5. Система предлагает кнопку "Проверить продукт" для поиска по списку

**Альтернативные сценарии:**
- **п.5: Пользователь вводит продукт вручную** → Система ищет по базе и возвращает результат
- **п.5: Продукт не найден в базе** → Система рекомендует уточнить у ветеринара

**Постусловия:** Пользователь ознакомлен со стоп-листом для своего питомца

## UC-006: Обновление веса питомца

| Элемент | Описание |
|---|---|
| **Название** | UC-006. Обновление веса и корректировка рациона |
| **User Story** | US-010: Как владелец питомца, я хочу вносить вес и видеть динамику |
| **Участники** | Основной: Владелец питомца. Вторичные: PetFeed Backend |
| **Предусловия** | Профиль питомца создан, есть хотя бы одна запись веса |
| **Ограничения** | Не более одной записи веса в день |
| **Триггер** | Пользователь нажимает "Обновить вес" или приходит еженедельное напоминание |

**Основной сценарий:**
1. Система отображает последнее зафиксированное значение веса и дату
2. Пользователь вводит новый вес
3. Система сохраняет запись с текущей датой
4. Система сравнивает с предыдущим значением и выводит динамику (+/-%)
5. Если вес изменился более чем на 5% — система пересчитывает нормы питания
6. Система уведомляет об изменении рациона и предлагает обновить рекомендации

**Альтернативные сценарии:**
- **п.2: Вес за пределами нормы для вида** → Система выводит предупреждение и рекомендует ветеринара
- **п.5: Резкое снижение/набор веса (>10%)** → Система добавляет срочное предупреждение

**Постусловия:** Новое значение веса сохранено, рацион скорректирован при необходимости

## UC-007: Взаимодействие с AI-ассистентом

| Элемент | Описание |
|---|---|
| **Название** | UC-007. Вопрос AI-ассистенту по питанию |
| **User Story** | US-014: Как владелец питомца, я хочу задать вопрос о питании и получить быстрый ответ |
| **Участники** | Основной: Владелец питомца. Вторичные: PetFeed Backend, LLM API (Claude / GPT) |
| **Предусловия** | Профиль питомца создан |
| **Ограничения** | Не более 10 запросов в день для бесплатного тарифа |
| **Триггер** | Пользователь нажимает "Задать вопрос" или пишет произвольный текст |

**Основной сценарий:**
1. Пользователь задаёт вопрос в свободной форме
2. Система добавляет контекст профиля питомца к запросу
3. Система отправляет запрос в LLM API
4. LLM формирует ответ с учётом вида, породы, возраста и веса питомца
5. Система выводит ответ пользователю
6. Система добавляет дисклеймер: "Не является ветеринарной консультацией"
7. Система предлагает связанные действия (обновить рацион, проверить стоп-лист)

**Альтернативные сценарии:**
- **п.3: LLM API недоступен (503)** → Система отвечает из кэша похожих вопросов или сообщает об ошибке
- **п.1: Вопрос не связан с питанием животных** → Система вежливо перенаправляет в контекст питомца
- **п.7: Лимит запросов исчерпан** → Система предлагает перейти на Premium

**Постусловия:** Ответ выведен, вопрос сохранён в истории диалога

## UC-008: Подключение партнёра-зоомагазина

| Элемент | Описание |
|---|---|
| **Название** | UC-008. Интеграция зоомагазина как партнёра |
| **User Story** | US-017: Как партнёр, я хочу интегрировать каталог в платформу, чтобы получать заказы |
| **Участники** | Основной: Партнёр (зоомагазин). Вторичные: Администратор, PetFeed Backend |
| **Предусловия** | Партнёр подписал договор и получил API-ключ |
| **Ограничения** | Только верифицированные партнёры с действующим договором |
| **Триггер** | Партнёр отправляет запрос на подключение через форму |

**Основной сценарий:**
1. Партнёр заполняет форму: название, сайт, API endpoint каталога
2. Администратор проверяет и подтверждает заявку
3. Система генерирует API-ключ для партнёра
4. Партнёр настраивает webhook для уведомлений о переходах
5. Система делает тестовый запрос к каталогу партнёра
6. Система подтверждает успешную интеграцию
7. Товары партнёра становятся доступны в рекомендациях

**Альтернативные сценарии:**
- **п.5: Каталог партнёра недоступен** → Система уведомляет администратора, интеграция приостановлена
- **п.2: Заявка отклонена** → Партнёр получает уведомление с причиной отказа

**Постусловия:** Партнёр интегрирован, его товары отображаются в рекомендациях
